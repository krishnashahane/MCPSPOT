## Prerequisites

Before installing MCPSpot, ensure you have the following prerequisites:

- **Node.js** 18+ (for local development)
- **Docker** (recommended for production)
- **pnpm** (for local development)

Optional for Smart Routing:

- **PostgreSQL** with pgvector extension
- **LLM Provider (OpenAI or compatible) API Key**

## Installation Methods

<Tabs>
  <Tab title="Docker (Recommended)">
    ### Docker Installation

    Docker is the recommended way to deploy MCPSpot in production.

    #### 1. Basic Installation

    ```bash
    # Pull the latest image
    docker pull krishnashahane/mcpspot:latest

    # Run with default settings
    docker run -d \
      --name mcpspot \
      -p 3000:3000 \
      krishnashahane/mcpspot:latest
    ```

    #### 2. With Custom Configuration

    ```bash
    # Create your configuration file
    cat > mcp_settings.json << 'EOF'
    {
      "mcpServers": {
        "fetch": {
          "command": "uvx",
          "args": ["mcp-server-fetch"]
        },
        "playwright": {
          "command": "npx",
          "args": ["@playwright/mcp@latest", "--headless"]
        }
      }
    }
    EOF

    # Run with mounted config
    docker run -d \
      --name mcpspot \
      -p 3000:3000 \
      -v $(pwd)/mcp_settings.json:/app/mcp_settings.json \
      krishnashahane/mcpspot:latest
    ```

    #### 3. With Environment Variables

    ```bash
    docker run -d \
      --name mcpspot \
      -p 3000:3000 \
      -e PORT=3000 \
      -e BASE_PATH="" \
      krishnashahane/mcpspot:latest
    ```

    #### 4. Docker Compose

    Create a `docker-compose.yml` file:

    ```yaml
    version: '3.8'
    services:
      mcpspot:
        image: krishnashahane/mcpspot:latest
        ports:
          - "3000:3000"
        volumes:
          - ./mcp_settings.json:/app/mcp_settings.json
        environment:
          - PORT=3000
          - BASE_PATH=""
          - REQUEST_TIMEOUT=60000
        restart: unless-stopped

      # Optional: PostgreSQL for Smart Routing
      postgres:
        image: pgvector/pgvector:pg17
        environment:
          POSTGRES_DB: mcpspot
          POSTGRES_USER: mcpspot
          POSTGRES_PASSWORD: mcpspot_password
        volumes:
          - postgres_data:/var/lib/postgresql/data
        ports:
          - "5432:5432"

    volumes:
      postgres_data:
    ```

    Run with:
    ```bash
    docker-compose up -d
    ```

    mcpspot

  <Tab title="Local Development">
    ### Local Development Setup

    For development, customization, or contribution:

    #### 1. Clone Repository

    ```bash
    # Clone the repository
    git clone https://github.com/krishnashahane/mcpspot.git
    cd mcpspot
    ```

    #### 2. Install Dependencies

    ```bash
    # Install dependencies with pnpm (recommended)
    pnpm install

    # Or with npm
    npm install

    # Or with yarn
    yarn install
    ```

    #### 3. Development Mode

    ```bash
    # Start both backend and frontend in development mode
    pnpm dev

    # This will start:
    # - Backend on http://localhost:3001
    # - Frontend on http://localhost:5173
    # - Frontend proxies API calls to backend
    ```

    #### 4. Build for Production

    ```bash
    # Build both backend and frontend
    pnpm build

    # Start production server
    pnpm start
    ```

    #### 5. Development Scripts

    ```bash
    # Backend only (for API development)
    pnpm backend:dev

    # Frontend only (when backend is running separately)
    pnpm frontend:dev

    # Run tests
    pnpm test

    # Lint code
    pnpm lint

    # Format code
    pnpm format
    ```

    <Note>
      On Windows, you may need to run backend and frontend separately:
      ```bash
      # Terminal 1: Backend
      pnpm backend:dev

      # Terminal 2: Frontend
      pnpm frontend:dev
      ```
    </Note>

  </Tab>

  <Tab title="Kubernetes">
    ### Kubernetes Deployment

    Deploy MCPSpot on Kubernetes with these manifests:

    #### 1. ConfigMap for Settings

    ```yaml
    apiVersion: v1
    kind: ConfigMap
    metadata:
      name: mcpspot-config
    data:
      mcp_settings.json: |
        {
          "mcpServers": {
            "fetch": {
              "command": "uvx",
              "args": ["mcp-server-fetch"]
            },
            "playwright": {
              "command": "npx",
              "args": ["@playwright/mcp@latest", "--headless"]
            }
          }
        }
    ```

    #### 2. Deployment

    ```yaml
    apiVersion: apps/v1
    kind: Deployment
    metadata:
      name: mcpspot
    spec:
      replicas: 1
      selector:
        matchLabels:
          app: mcpspot
      template:
        metadata:
          labels:
            app: mcpspot
        spec:
          initContainers:
            - name: prepare-config
              image: busybox:1.28
              command:
                [
                  "sh",
                  "-c",
                  "cp /config-ro/mcp_settings.json /etc/mcpspot/mcp_settings.json",
                ]
              volumeMounts:
                - name: config
                  mountPath: /config-ro
                  readOnly: true
                - name: app-storage
                  mountPath: /etc/mcpspot
          containers:
            - name: mcpspot
              image: krishnashahane/mcpspot:latest
              ports:
                - containerPort: 3000
              resources:
                requests:
                  cpu: 100m
                  memory: 128Mi
                limits:
                  cpu: 500m
                  memory: 512Mi
              env:
                - name: PORT
                  value: "3000"
                - name: MCPHUB_SETTING_PATH
                  value: /etc/mcpspot/mcp_settings.json
              volumeMounts:
                - name: app-storage
                  mountPath: /etc/mcpspot
          volumes:
            - name: config
              configMap:
                name: mcpspot-config
            - name: app-storage
              emptyDir: {}
    ```

    #### 3. Service

    ```yaml
    apiVersion: v1
    kind: Service
    metadata:
      name: mcpspot-service
    spec:
      selector:
        app: mcpspot
      ports:
      - port: 80
        targetPort: 3000
      type: ClusterIP
    ```

    #### 4. Ingress (Optional)

    ```yaml
    apiVersion: networking.k8s.io/v1
    kind: Ingress
    metadata:
      name: mcpspot-ingress
      annotations:
        nginx.ingress.kubernetes.io/proxy-buffering: "off"
    spec:
      rules:
      - host: mcpspot.yourdomain.com
        http:
          paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: mcpspot-service
                port:
                  number: 80
    ```

    Deploy with:
    ```bash
    kubectl apply -f mcpspot-configmap.yaml
    kubectl apply -f mcpspot-deployment.yaml
    kubectl apply -f mcpspot-service.yaml
    kubectl apply -f mcpspot-ingress.yaml
    ```

## Smart Routing Setup (Optional)

Smart Routing provides AI-powered tool discovery using vector semantic search.

### Prerequisites

1. **PostgreSQL with pgvector**
2. **LLM Provider (OpenAI or compatible) API Key**

### Database Setup

<Tabs>
  <Tab title="Docker PostgreSQL">
    ```bash
    # Run PostgreSQL with pgvector
    docker run -d \
      --name mcpspot-postgres \
      -e POSTGRES_DB=mcpspot \
      -e POSTGRES_USER=mcpspot \
      -e POSTGRES_PASSWORD=your_password \
      -p 5432:5432 \
      pgvector/pgvector:pg16
    ```
  </Tab>

  <Tab title="Existing PostgreSQL">
    If you have an existing PostgreSQL instance:

    ```sql
    -- Connect to your PostgreSQL instance
    -- Create database
    CREATE DATABASE mcpspot;

    -- Connect to the mcpspot database
    \c mcpspot;

    -- Enable pgvector extension
    CREATE EXTENSION IF NOT EXISTS vector;
    ```

  </Tab>

  <Tab title="Cloud PostgreSQL">
    For cloud providers (AWS RDS, Google Cloud SQL, etc.):

    1. Enable the pgvector extension in your cloud provider's console
    2. Create a database named `mcpspot`
    3. Note down the connection details

  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Docker Issues">
    **Port already in use:**
    ```bash
    # Check what's using port 3000
    lsof -i :3000

    # Use a different port
    docker run -p 8080:3000 krishnashahane/mcpspot
    ```

    **Container won't start:**
    ```bash
    # Check container logs
    docker logs mcpspot

    # Run interactively for debugging
    docker run -it --rm krishnashahane/mcpspot /bin/bash
    ```

  </Accordion>

  <Accordion title="npm Installation Issues">
    **Permission errors:**
    ```bash
    # Use npx instead of global install
    npx @krishnashahane/mcpspot

    # Or fix npm permissions
    npm config set prefix ~/.npm-global
    export PATH=~/.npm-global/bin:$PATH
    ```

    **Node version issues:**
    ```bash
    # Check Node version
    node --version

    # Install Node 18+ using nvm
    nvm install 18
    nvm use 18
    ```

  </Accordion>

  <Accordion title="Network Issues">
    **Can't access dashboard:**
    - Check if MCPSpot is running: `ps aux | grep mcpspot`
    - Verify port binding: `netstat -tlnp | grep 3000`
    - Check firewall settings
    - Try accessing via `127.0.0.1:3000` instead of `localhost:3000`

    **AI clients can't connect:**
    - Ensure the endpoint URL is correct
    - Check if MCPSpot is behind a proxy
    - Verify network policies in Kubernetes/Docker environments

  </Accordion>

  <Accordion title="Smart Routing Issues">
    **Database connection failed:**
    ```bash
    # Test database connection
    psql $DB_URL -c "SELECT 1;"

    # Check if pgvector is installed
    psql $DB_URL -c "CREATE EXTENSION IF NOT EXISTS vector;"
    ```

    **Embedding service errors:**
    - Verify LLM provider API key is valid
    - Check internet connectivity
    - Monitor rate limits

  </Accordion>
</AccordionGroup>
